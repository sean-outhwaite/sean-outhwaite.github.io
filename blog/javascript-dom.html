The Three Amigos

That's where JavaScript comes in. With JS you can make your dreams come true, buttons now do things, data can now be stored and transformed, the web is your oyster.

If the web were a superhero, HTML would be their core (morals & personality), CSS their costume, and JS their superpower.

World DOMination

The DOM is a representation of all of the content of a webpage, handily provided in the format of a javascript object. It allows you to access and change the content of a page programatically.

You can think of the DOM like a tree, parts of the webpage split and branch down into different nodes.

It‚Äôs important to note that the DOM is a living object (like a tree), any changes you make to the DOM will apply to the HTML/CSS and if the HTML or CSS is changed, the DOM will reflect that.

Although it may seem like the DOM is part of javascript, it‚Äôs actually provided by the browser along with an API that lets javascript (or any programming language) interact with it.

Let's Get Func-y

Functions are essentially re-usable pieces of code that do something.

That could be adding numbers together, updating some text somewhere, looping through an array, the possibilities are endless! Functions can even call other functions, which in turn can call other functions, which in turn can call other functions, which... sorry I'll stop now.

One of the core principles of clean code is DRY - don‚Äôt repeat yourself (ignore the previous sentence).

Functions exist to help us adhere to that. Basically, whenever there‚Äôs something you are going to want to do multiple times in your code - you should make it a function.

Rather than repeating the code every single time you want to do that thing, you can just call the function as needed. And if you ever need to change how that functionality works, you now only need to do it in one place instead of everywhere you're doing that thing.
