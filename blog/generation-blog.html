Blog Generation

How to turn your laziness into productivity

29 Sep 2025

Laying the Foundations

One of the most tedious things I found about the foundations content at Dev Academy was creating all of the HTML for my blog posts.

I would spend time writing and editing my content until I was happy with it, then have to spend even more time chopping up that content and nicely wrapping each piece in the relevant HTML tags.

There has got to be a better way

Instead of spending a small amount of time doing this for each blog post, I decided to spend a large-ish amount of time building a solution for myself so that I would never have to do this again (although maybe I already never had to do this again as now I have finished foundations I have no obligation to continue to update this blog)

I wanted to build something that would allow me to write my content just as I would in a text editor and then automatically give me that content in beautiful HTML structure.

Introducing... the Blog Generator

I built a page that had inputs for all (well most...) of the things I use on my blog posts and not only gives me the full HTML I need but also shows me a preview of what that content looks like.

All of the top content (the main and sub headings + date) can be edited and the preview + output will live update.

Section headings and paragraphs will just continue to be appended as they're added.

Trials & Tribulations

My first version of the generator actually worked pretty well, but for all elements it would just append them to the bottom of the preview + output (from here on out I will refer to this as prevput).

But this meant that you had to add the elements in the order they were going to appear on the page and that you could add multiples of some elements (e.g. main heading) where there should really only be one.

So rather than just appending those things to the prevput, I decided to store them in an object. This allowed me to always place them in the correct location for my desired page structure and ensured that only one of each of those elements was included - as an added bonus it also meant that you could edit the content of those elements.

This also provided a solution for me to add the rest of my boilerplate HTML. I had only included what would go in the main section of the page in my HTML output, because I didn't want content to be placed in the wrong place.

But I was now able to add the top and bottom of my HTML to the object and always output it in the correct order, so the final output can be pasted into a completely empty HTML file.

Dogfooding

As I began writing this blog using the generator, I noticed some annoying things.

I had placed the output directly underneath the preview, with the input fields underneath that. This meant that as more content was added, the inputs got pushed further and further down the page and it was quite hard to distinguish where the preview ended and the output began.

To make this better, I moved the output to the bottom as I don't really need to see the HTML as I'm writing the content. I also added a border to the preview so it's a bit more contained.
